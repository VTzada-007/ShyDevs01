## Descrição do caso escolhido

A ShyDevs Studio é um estúdio independente focado no desenvolvimento de experiências para a plataforma Roblox. O projeto consiste na criação de um site institucional com o objetivo de apresentar o estúdio, divulgar seus grupos oficiais, disponibilizar canais de contato e apresentar seus colaboradores.

---

## Necessidade identificada

A ShyDevs Studio não possuía uma página centralizada para apresentar suas informações de forma organizada. Os usuários precisavam acessar diferentes plataformas para encontrar grupos, contatos e informações sobre o estúdio.

O site foi desenvolvido para reunir essas informações em um único ambiente, facilitando a comunicação com jogadores, membros da comunidade e possíveis colaboradores.

---

## Público-alvo

O público-alvo do projeto inclui:

* Jogadores da plataforma Roblox;
* Membros dos grupos da ShyDevs Studio;
* Interessados em acompanhar os projetos do estúdio;
* Possíveis colaboradores e desenvolvedores;
* Usuários que desejam entrar em contato com a equipe.

---

## Objetivo do site

O principal objetivo do site é apresentar a ShyDevs Studio e facilitar o acesso às suas principais informações.

A página foi desenvolvida para permitir que o visitante:

* Conheça o estúdio;
* Acesse os grupos oficiais;
* Visualize os colaboradores;
* Entre em contato com a equipe.

---

## Processo de desenvolvimento

O projeto foi desenvolvido seguindo as seguintes etapas:

1. Definição do caso real a ser representado;
2. Levantamento das informações necessárias;
3. Criação do wireframe da página;
4. Estruturação do conteúdo utilizando HTML semântico;
5. Desenvolvimento da identidade visual utilizando CSS;
6. Implementação da responsividade para diferentes dispositivos;
7. Aplicação de práticas básicas de acessibilidade;
8. Desenvolvimento da funcionalidade interativa em JavaScript;
9. Testes de funcionamento e navegação;
10. Publicação do projeto.

## Principais decisões do projeto

### Estrutura HTML

Foram utilizados elementos semânticos para melhorar a organização e acessibilidade da página:

* header
* nav
* main
* section
* footer

### Organização Visual

Foi utilizada uma combinação de tons escuros e azul-claro para transmitir uma identidade moderna e relacionada ao universo de tecnologia e jogos digitais.

### Responsividade

O layout foi adaptado para diferentes tamanhos de tela através de Media Queries.

Em dispositivos móveis:

* O menu é reorganizado;
* Os grupos passam a ser exibidos verticalmente;
* Os colaboradores são empilhados;
* O conteúdo permanece legível sem rolagem horizontal.

### Acessibilidade

Foram aplicadas boas práticas como:

* Uso de atributo alt em todas as imagens;
* Hierarquia correta de títulos;
* Contraste adequado entre texto e fundo;
* Navegação simples e intuitiva.

---

## Funcionalidade em JavaScript

Foi implementado um sistema de expansão e recolhimento de conteúdo na seção de grupos.

Ao clicar no botão "Mostrar Mais", o usuário visualiza informações adicionais sobre o grupo selecionado.

Ao clicar novamente, as informações são ocultadas.

Essa funcionalidade foi escolhida para melhorar a organização visual da página e evitar excesso de conteúdo na tela.

---

## Uso de Bootstrap

O projeto utiliza Bootstrap para complementar a estilização dos componentes interativos.

Recursos utilizados:

* Botões Bootstrap (`btn btn-primary`);
* Sistema de containers responsivos;
* Classes utilitárias para espaçamento.

A utilização do Bootstrap contribuiu para a padronização visual e para o desenvolvimento responsivo da interface.
